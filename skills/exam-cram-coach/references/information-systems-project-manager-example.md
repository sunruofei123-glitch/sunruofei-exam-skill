# Example: Information Systems Project Manager Final-Week Sprint

Use this as an example pattern, not as a fixed syllabus for every exam.

## Learner Situation

- Exam: 信息系统项目管理师
- Remaining time: about five days
- Stronger area: multiple choice
- Weak areas: essay, case calculations, process definitions, outputs, tools, performance domains
- Daily limit: about two hours, with occasional user-driven overrun

## Source Materials Used

Ask the learner to upload or identify materials like these before building the plan:

- sprint summary PDF for high-frequency theory
- case calculation practice PDF
- operations research practice PDF
- case analysis practice PDF
- theory must-memorize handout
- learner's own mock answers, essay paragraphs, and calculation attempts

In this example, the coach prioritized the must-memorize handout for theory wording and used practice PDFs for calculation/case drills. When the learner pointed out that theory handout wording or mnemonics were missing, the coach updated the plan and extracted a compact reference.

## Study Strategy

1. Build one reusable project background:
   - 智慧航站楼综合运行管理平台
   - management cockpit, terminal operations platform, mobile app
   - many stakeholders, external integration, change requests, airport intranet constraints

2. Drill theory in "five-piece" form:
   - process name
   - definition
   - purpose
   - high-frequency tools
   - core outputs

3. Track weak points aggressively:
   - wrong formulas
   - confused terms
   - missing outputs
   - essay paragraphs that sound like notes instead of exam writing

4. Use active recall:
   - learner answers first
   - coach grades
   - coach records weak point
   - learner re-answers only the missed items

5. Reserve the final day for red-line review:
   - no broad new content
   - formulas
   - standard terms
   - essay outlines
   - repeated weak points

## Useful Drills From This Example

### Earned Value

Ask:

```text
SV, CV, SPI, CPI?
Typical EAC?
Non-typical EAC?
EAC when both cost and schedule performance affect remaining work?
ETC?
VAC?
```

Red-line formulas:

```text
SV = EV - PV
CV = EV - AC
SPI = EV / PV
CPI = EV / AC
EAC = AC + (BAC - EV)
EAC = BAC / CPI
EAC = AC + (BAC - EV) / (CPI * SPI)
ETC = EAC - AC
VAC = BAC - EAC
```

### Schedule Crashing

Ask for:

```text
unit crash cost
first activity to crash
second activity to crash
total additional cost
new critical path
```

Common correction:

```text
When multiple critical paths appear, reduce a common activity or reduce all critical paths.
```

### Performance Domains

Ask:

```text
Eight performance domains?
Project Work performance domain eight key points?
Uncertainty performance domain four response methods?
Two expected-outcome/check-indicator pairs for each domain?
```

Red-line examples:

```text
Project Work: project processes, constraints, focus on work processes and capability, communications and engagement, physical resources, procurements, new work and changes, continuous improvement.
Uncertainty responses: gather information, prepare for multiple outcomes, set-based design, build resilience.
```

### Configuration and Change

Ask:

```text
Configuration management six processes?
Three configuration libraries?
Configuration audit functions?
Change workflow steps?
Case-version change workflow?
```

Common correction:

```text
Problem questions must be answered with "未..., 没有..., 擅自..., 导致..." rather than "应该...".
```

### Essay Coaching

For each candidate topic, ask the learner to write an outline first:

```text
1. project background
2. theory goals / process list
3. how I planned or executed
4. how I monitored and corrected
5. lesson learned
```

High-priority essay topics from this example:

- 论信息系统项目的规划绩效域
- 论信息系统项目的度量绩效域
- 论信息系统项目的范围管理

## Coaching Tone

- Be direct and specific.
- Keep the learner moving.
- Do not over-teach what is already correct.
- Record recurring mistakes and ask them again tomorrow.
- If the learner says "next", move on and keep the skipped item in the final review queue.
